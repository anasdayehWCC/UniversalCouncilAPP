UI/UX critique and implementation planning for MinutePlatform app based on 9 screenshots across 6 views (Home, My Notes, Recording, Persona Selector, Note Detail/AI Panel, Insights).

## What Was Done
- Analysed all screenshots in `Images/` directory

- Identified 10 concrete UI/UX issues with root causes traced to source files
- Created comprehensive implementation plan: `docs/plans/2026-03-28-ui-ux-comprehensive-redesign.md`

## Key Issues Found

| ID | Issue | Root Cause |
|----|-------|-----------|
| A | My Notes: dark cards on light background | Hardcoded dark bg colors in MeetingCard, not using `bg-card` token |
| B | Home: inconsistent icon fill/colour on action cards | No shared icon treatment component; each card hand-rolled differently |
| C | AI panel: 40% width + heavy backdrop-blur blocks content | Panel uses overlay + backdrop-filter instead of sidebar push pattern |
| D | Insights: metric count badge floats ABOVE icon (inverted hierarchy) | Custom floating-badge-above-icon layout, not a standard metric card |
| E | Review queue: "Overdue by 11742h" raw hours, badge overload | No time-formatting helper; no badge count cap |
| F | Persona selector: all 6 cards look identical | No role-type color differentiation applied |
| G | Header breadcrumb truncates mid-word | No `shrink-0` on status pill; breadcrumb has no max-width/tooltip |
| H | Recording: large empty waveform box before recording starts | Layout is waveform-first instead of mic-button-first |
| I | All-caps section labels inconsistent across components | No canonical `.label-section` utility class |
| J | Insights empty state shows bare zeros | No empty-state component when data is 0 |

## Decisions Made
- No new libraries introduced — all fixes within existing token/Tailwind system
- AI panel fix: push-content pattern (adds `mr-80` to main) rather than overlay
- Persona cards: role-type drives border/badge colour (blue=practitioner, amber=manager, purple=admin, teal=housing)
- Overdue formatting: `formatOverdue(hours)` helper converts to d/wk human-readable
- Badge cap: max 3 visible per card, `+N` overflow badge
